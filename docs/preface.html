Preface

欢迎来到 R Packages 的第二版！如果您熟悉第一版，本前言描述了主要变化，以便您可以将阅读重点放在新领域。

此版本有几个主要目标：

  • 更新以反映 devtools 包中的变化，特别是它 “conscious uncoupling” 成一组更小、更集中的包。

  • 扩大工作流程和流程的覆盖范围，同时展示构成 R 包的所有重要移动部件。

  • 涵盖全新主题，例如 package websites 和 GitHub Actions。

所有内容均已完全修改和更新。许多章节都是新增或重新组织的，其中一些章节已被删除：

  • 新的 Chapter 1，“The Whole Game”，预览了整个 package 的开发过程。

  • 新的 Chapter 2，“System setup”，是从之前的简介中删除的，并获得了更多详细信息。

  • 以前称为 “Package structure” 的章节已扩展并分为两章，一章介绍包结构和状态（Chapter 3），另一章介绍工作流程和工具（Chapter 4）。

  • 新的 Chapter 5，“The package within”，演示了如何从数据分析脚本中提取可重用逻辑并将其放入 package 中。

  • Chapter 6，“R code” 中的 “Organising your functions” 和 “Code style” 部分已被删除，取而代之的是在线风格指南，https://style.tidyverse.org/。该样式指南与新的 styler 包 (Müller and Walthert 2018) 配对，可以自动应用许多规则。

  • 测试的覆盖范围已扩展为三章：Chapter 13 为 testing basics，Chapter 14 为 test suite design，Chapter 15 为 various advanced topics。

  • 围绕 NAMESPACE 文件和依赖关系的材料已重新组织为两章：Chapter 10 提供了思考依赖关系的技术背景，Chapter 11 提供了在不同设置中使用不同类型依赖关系的实践说明。

  • 新的 Chapter 12，“Licensing”，将之前有关许可的内容扩展为自己的章节。

  • 关于 C/C++ 的章节已被删除。它没有足够的信息来发挥作用，自本书第一版以来，出现了其他更好的学习资源。

  • “Other components” 一章已被删除。

  • 关于 Git/GitHub 的章节已围绕软件开发实践这一更一般的主题进行了重新构建（Chapter 20）。这不再包括基本任务的分步说明。自第一版以来，Git/GitHub 的使用呈爆炸式增长，伴随着 R 通用和特定学习资源的爆炸式增长（e.g. the website Happy Git and GitHub for the useR）。 Git/GitHub 仍然在整本书中占据突出地位，尤其是在 Chapter 20。

  • 非常短的 inst 章节已合并到 Chapter 8 中，所有其他目录在特定情景中可能很重要，但并非对所有包都至关重要。

Conventions

在本书中，我们用 fun() 来引用函数，用 var 来引用变量和函数参数，用 path/ 来引用路径。

较大的代码块混合输入和输出。输出带有注释，因此如果您有本书的电子版本，例如 https://r-pkgs.org，您可以轻松地将示例复制并粘贴到 R 中。输出注释看起来像 #> 以将其与常规注释区分开。
